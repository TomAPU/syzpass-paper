\begin{abstract}
Modern Linux kernel exploit development hinges on identifying powerful exploit primitives, i.e., unintended memory operations that can corrupt security-critical kernel objects. Unfortunately, these primitives are often entangled with additional undesirable memory operations, or side effects (e.g., dereferencing invalid pointers), which can trigger crashes and derail exploit attempts. Although side effects are well known in manual exploit development, they remain an under-modeled obstacle in automated exploit guidance.

In this paper, we model side-effect handling as primitive-aware guidance for kernel exploit construction. SyzPass identifies side effects and reports guidance using high-sensitivity context exploration, symbolic-execution-guided path exploration, and object-layout matching. Starting from 342 real-world Linux kernel bugs, SyzPass obtains strict primitive-aware results for 97 aggregate records, covering 781 counted primitive sites. Under the portable object-matching baseline, 15 sites satisfy our candidate predicate: one is already clean and 14 are SyzPass-attributed neutralization candidates. Under a stronger pointer-materialization model, 276 additional pointer-related sites become candidates, exposing how modern disclosure and staging capabilities change primitive usability. Three expert-assisted privilege-escalation case studies validate selected guidance: one exercises a counted object-matching neutralization candidate, while two validate path and constraint guidance outside the strict aggregate.

\end{abstract}
